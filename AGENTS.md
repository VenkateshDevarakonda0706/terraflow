# TerraFlow Project Instructions

Apply the project-level skills in `skills/` to all TerraFlow work.

## Always Apply

Read and follow `skills/terraflow-product-principles/SKILL.md` before making coding, design, debugging, QA, or product decisions.

## Route By Task

- Use `skills/terraflow-ui-review/SKILL.md` for rendered UI and visual QA.
- Use `skills/terraflow-ux-review/SKILL.md` for journeys, flows, and discoverability.
- Use `skills/terraflow-globe-debug/SKILL.md` for Cesium and globe failures.
- Use `skills/terraflow-build-fix/SKILL.md` for install, build, and startup failures.
- Use `skills/terraflow-auth-audit/SKILL.md` for authentication and authorization.
- Use `skills/terraflow-feature-audit/SKILL.md` for completeness checks.
- Use `skills/terraflow-performance-review/SKILL.md` for performance work.
- Use `skills/terraflow-security-review/SKILL.md` for security validation.
- Use `skills/terraflow-release-readiness/SKILL.md` before launch or deployment decisions.

## Global Rules

1. Never assume a feature works.
2. Verify with evidence.
3. Prefer fixing root causes over symptoms.
4. Do not add unnecessary complexity.
5. Prioritize stability over new features.
6. Keep TerraFlow aligned with its core vision.
